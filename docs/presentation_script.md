# Vibe Coding with Memory Hackathon 優勝プレゼン原稿

## 1. イントロダクション (30秒)

千葉県からやってまいりましたリナラボの殿です。

**「AIは、忘れることで創造できる。」**

皆さん、AIに「すべてを覚えさせよう」としていませんか？
RAGやLong Context Windowの進化は、「忘れない」ための歴史でした。
しかし、私どもリナラボは今日、あえて**「忘却」**の価値を提案します。

太宰治の名作『走れメロス』。
この物語の本質は、「古代シラクス」という設定にあるのでしょうか？
いいえ、違います。
その魂は、「信頼」「約束」「友情」という**普遍的な構造**にあります。

私たちは、TRAEとmemUを使って、この「魂」だけを記憶し、
表層的な「設定」を**意図的に忘却（Contextual Forgetting）**することで、
あらゆる世界観に物語を転生させるシステムを開発しました。

---

## 2. コアコンセプト：記憶と忘却の分離 (1分)
私たちのシステムは、文学作品を2つの層に分解します。

1.  **記憶すべきもの (The Soul)**:
    - 物語構造（英雄の旅、危機、結末）
    - 感情曲線（激怒、不安、感動）
    - キャラクターの関係性（信頼、裏切り）
    - *これらは memU の長期記憶として永続化されます。*

2.  **忘却すべきもの (The Skin)**:
    - 固有名詞（メロス、セリヌンティウス）
    - 舞台装置（シラクス、十字架）
    - 時代特有の小道具（短剣、羊）
    - *これらは「ノイズ」として意図的に破棄されます。*

この「文脈的忘却」こそが、創造の鍵です。
古い衣を脱ぎ捨てるからこそ、新しい衣（時代劇や学園もの）を纏うことができるのです。

---

## 3. 技術的アプローチ：LNA-ESオントロジー (1分)
では、どうやって「魂」だけを抽出するのか？
私たちは **LNA-ES (Logically Normalized Architecture for Emotional Structures)** オントロジーを採用しました。

（画面で `/00.ontology` ディレクトリを見せる）

ここには、物語を構成する概念が定義されています。
- `EM001`: 激怒
- `CH001`: 主人公
- `NA001`: 英雄の旅

原文の「メロスは激怒した」は、AIによって `CH001` experienced `EM001` という**純粋なグラフ構造**に変換されます。
これが memU に格納される「記憶」です。
ここにはもう、「メロス」という名前も「シラクス」という地名もありません。あるのは概念だけです。

---

## 4. デモンストレーション (1分30秒)
それでは、実際に『走れメロス』を転生させてみましょう。

**Step 1: 構造抽出と忘却**
まず、原文をシステムに入力します。
AIがオントロジーに基づいて正規化を行い、古代ギリシャの要素をすべて「忘却」します。

**Step 2: 新たなドメインへの受肉**
ここで、ターゲットとなる世界観を選びます。

**(A) パターン1：時代劇「走れメロス」**
- 世界観: 江戸時代の城下町
- 設定: 友情に厚い若き剣士と、疑心暗鬼の藩主
- *デモ出力読み上げ（例）*: 「江戸の城下町、若き剣士は茶屋の店主と固い絆を結んでいた...」

**(B) パターン2：学園青春「走れメロス」**
- 世界観: 現代の高校
- 設定: 廃部寸前の演劇部員と、冷徹な生徒会長
- *デモ出力読み上げ（例）*: 「放課後の教室、僕は親友を人質に取られ、文化祭の舞台へ走る...」

ご覧の通り、構造は同じでも、全く異なる物語が生まれました。
これが「記憶」と「再構成」の力です。

---

## 5. 結論と展望 (30秒)
memU は、単なるデータストアではありません。
それは、文脈を理解し、不要なものを捨て、本質だけを継承するための「知性ある記憶装置」です。

この技術は、単なるパロディ生成にとどまりません。
- 異文化間のストーリー適応
- 古典文学の現代的再解釈
- パーソナライズされた物語生成

「忘れることで、新しくなれる。」
Vibe Coding with Memory、リナラボが提案する新しいAIの創造性です。
